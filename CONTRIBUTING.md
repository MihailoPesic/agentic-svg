# Contributing

Thanks for taking the time to work on agentic-svg. It's a small, focused
codebase, so keep changes tight and self-contained.

## Requirements

- Node.js 20 or newer (24 recommended)
- npm

No global tools needed. Native deps (`sharp`, `@resvg/resvg-js`,
`@neplex/vectorizer`) ship prebuilt binaries, so `npm ci` should just work.

## Setup

```sh
npm ci
```

## Running

```sh
# Convert an image from the CLI
node src/cli.js <input.png> out.svg --quality high

# Start the HTTP server + web UI
npm run server
```

The web UI lives in `web/` and is served by the server.

## Tests

Tests use the built-in Node test runner — no extra framework.

```sh
npm test            # runs node --test over test/
node --test test/core.test.js   # a single file
```

Add tests under `test/` as `*.test.js`. Keep them fast and deterministic;
fixtures live in `fixtures/` and are generated by the scripts in `scripts/`.

## Project layout

```
src/
  core/        conversion engine
    pipeline.js   entry point: convertImage(input, { quality })
    converge.js   iterative refinement loop
    image.js      raster.js shapes.js render.js trace.js
    gradient.js   saliency.js metrics.js optimizer.js classify.js
  cli.js       command-line interface
  server/      HTTP server
web/           browser front-end
fixtures/      test images
scripts/       fixture generators and benchmarks
test/          node --test suites
```

The public surface is `convertImage` from `src/core/pipeline.js` and the
`converge` loop. Don't break those signatures without a good reason.

## Style

- ESM only (`"type": "module"`), modern Node, no transpiler.
- Terse and practical. Comment only where the reason isn't obvious from the code.
- Match the surrounding code rather than reformatting unrelated lines.

## Pull requests

- One logical change per PR.
- Run `npm test` before opening it. CI runs the same on Node 20 and 22.
- Describe what changed and why in the PR body.
